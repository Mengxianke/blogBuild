RABC权限控制 ​

ToB的项目中，不同的角色能够访问到的权限不同。权限体现在用户访问url地址, 菜单栏，以及某些特定展现的区域上。 也就是接口权限，按钮权限，菜单权限，路由权限。 针对这些涉及到权限控制的地方，我们梳理与回顾项目中用到权限控制与管理的方法。动态路由, 路由守卫, 指令这三种方法。

动态路由 ​

动态路由是由后端返回路由结构，与前端维护的路由表进行匹配。筛选出符合与当前用户权限匹配的路由。 在通过渲染的方式，将筛选出来的路由转化成菜单栏。这种方案是通过后端主导，前端配合的一种方式。

  1. 前端维护路由表
    一般来讲，会维护两个路由表结构。 第一个是所有用户都会有权限的路由。

  1. 筛选出属于用户权限的路由表
js
/**
 *
 * @param  {Array} userRouter 后台返回的用户权限json
 * @param  {Array} allRouter  前端配置好的所有动态路由的集合
 * @return {Array} realRoutes 过滤后的路由
 */
export function filerAsyncRoutes(userRouter = [], allRouter = []) {
    var realRoutes = []
    // 遍历前端配置表
    allRouter.forEach((v, i) => {
        userRouter.forEach((item, index) => {
            if (item.name === v.meta.name) {
                if (item.children && item.children.length > 0) {
                    v.children = filerAsyncRoutes(item.children, v.children)
                }
                realRoutes.push(v)
            }
        })
    })
    return realRoutes
}

路由守卫 ​

路由守卫是由前端主导，借助前端框架，在跳转前进行拦截的主要方式。

指令 ​

指令控制按钮Button的显示。屏蔽掉用户的功能。 并不是所有用户都可以对内容有增删的权限。获取到当前用户的身份， 在相应的按钮上绑定指令，给指令传入当前按钮所需要的权限。根据身份和权限进行匹配， 如果当前用户没有操作按钮的权限，通过指令访问按钮元素的父节点，从父节点中，将按钮进行删除。

vue
<div>
    <el-button v-hasPermi="[....]"></el-button>
</div>
