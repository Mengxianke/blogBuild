Vue的ViewModel机制 ​

Vue实现从Model到View视图的更新，需要回答如下两个问题。

  1. 第一个问题是Vue是如何做依赖收集的?
  2. 第二个问题是Vue如何根据依赖的变化更新视图?

Vue 响应式依赖类图 ​

Vue的响应式依赖离不开重要的三个类: 一个是Dep， 一个是Wather类, 以及Observer类。 Watcher类负责更新视图。 Dep类是管理依赖类，负责通知视图更新。 当响应式数据状态发生更新的时候，需要通知到其影响到的Watcher类进行视图更新，这种消息的通知采用观察者模式，Dep类是被观察者，Watcher类属于观察者，两者属于多对多的关联关系。Observer类持有Dep管理类，Observer类负责将观察者对象注册到被观察者的观察者列表中。

Dep与Watcher之间的关系为何是多对多的关联关系? "响应式"数据在Vue中体现在模板，计算属性，watch监听器中。 它们的变化会引起视图的变化。因此Vue将更新视图的watcher类进行了细化。分别定义了 render watcher, 负责收集模板中的依赖。 computed watcher，负责收集计算属性中的依赖。 wach watcher, 负责收集监听器中的依赖关系。一个响应式的数据，可以用在模板中，也可以在computed中，也可在watch中监听。 因此，一个dep实例类可以对应多个watcher实例，反过来一个watcher可以对应多个dep管理类。两者是多对多的关联关系。

建立多对多的关联关系 ​

从观察者设计模式的Clinet端出发。 观察者模式的Clinet端负责将观察对象注册到被观察者列表中。Observer类负责注册的功能。数据会通过Observer类转变为"响应式"数据. Observer遍历对象，给每一个key对应的数据定义响应式。 defineReactive中，创建一个管理依赖的类。 定义响应式是通过Object.defineProerpty进行拦截。getter中调用dep.depend()方法进行注册。

js
class Observer {
    walk (obj: Object) {
        const keys = Object.keys(obj);
        for (let i = 0; i < keys.length; i++) {
            defineReactive(obj, keys[i]);
        }
    }
}

        configurable: true,

                if (isArray(value)) {

            }
            return isRef(value) && !shallow ? value.value : value

      // .....
    })
}

但getter拦截器中并没有明确指明，将哪些观察者注册到Dep实例的观察者列表中。 访问数据时，"外部"当前已经明确观察者对象，这个观察者对象是存放在Dep的静态变量target变量中。 调用dep的depend方法，并没有直接往dep的观察者列表中直接注册观察者watcher实例(target)。 为了维护dep和watcher的关联关系，将调用watcher的addDep(this)方法。最终watcher的addDep方法会调用dep中的addSub方法 进行注册。

观察者模式的建立过程为: 创建watcher实例，并调用watcher的get方法，将当前实例变为一个全局的观察者。 访问数据的过程中，通过Object.defineProperty进行访问拦截。调用get方法。触发Dep实例中的depend方法， depend方法中，调用Watcher的addDep方法，入参为当前dep实例。最终在watcher中调用dep实例的addSub进行注册， 以及dep和watcher双方建立关系。

依赖收集 ​

上文中未解决的问题是，定义的响应式数据何时进行访问? 因为只有访问了数据被getter进行拦截，才能 确立观察者和被观察者之间的关系。watcher实例又是何时被创建?
短暂回顾一下，我们已经知道Vue的数据驱动UI是通过观察者这一设计模式实现的。 一切的根源是通过创建watcher类开始的。创建watcher类时候会执行watcher的get()方法。此方法中 通过pushTarget()明确观察者实例，以及调用某个getter函数，这个getter函数会触发对响应式数据的访问。

我们来思考一下Vue2的响应式数据定义在哪里。在computed中，watch中，以及模板中。 因此针对这三种数据，Vue分别定义出计算属性watcher(computedWatcher), 监听器watcher(userWatcher)以及渲染更新的DOM的watcher(render Watcher). 这三种watcher分别负责对响应式数据进行依赖收集。

依赖收集的触发时间 ​

Vue组件的初始化的顺序

js
initLifecycle(vm)
initEvents(vm)
initRender(vm)
callHook(vm, 'beforeCreate')
initInjections(vm)
**initState(vm)**
initProvide(vm)
callHook(vm, 'created')
if (vm.$options.el) {  
    vm.$mount(vm.$options.el)
}

在initState(vm)的过程中, 处理数据的顺序是

  1. initProps
  2. initMethods
  3. initData
  4. initComputed
  5. initWatch

在initData阶段中，将数据通过defineReactive定义为响应式。 在initState中调用initComputed方法，访问了定义在组件中的computed，在initWatch中 访问了在组件中定义的watch方法。 在$mount函数中，进行组件挂载，访问了模板上的数据。

因此这三种watcher的初始化的先后顺序是: computed > user > render.

依赖收集的过程 ​

在我们明确了依赖收集的触发时间后，即何时创建wathcer, 何时将数据转化成响应式变量。 我们将目光转移到如何触发的数据访问上。因为只有触发了数据访问才能触发后续的关系的建立。 我们在此审视一下wathcer类的，构造函数以及get()方法。watcher类中的构造函数中表明创建这个类的依赖。 它需要expOrFn这个函数。expOfFn会转化为getter函数，在构造函数中调用get()方法，会调用getter()方法。 这个getter()方法就是访问数据的时机。

            // 调用getter函数

Computed Watcher ​

comptued watcher收集定义在computed中的响应式变量。

js
export default {
    data() {
        return {
            msg: '';
        }
    },
    computed: {
        upperMsg() {
            return this.msg.toUpperCase();
        }
    }
}

computed watcher的构造是在Vue组件初始化的initState的initComputed过程中。 initComputed函数的入参是vm，vm为当前组件实例。 initComputed函数中遍历computed属性, 并获取getter函数。执行getter函数会访问变量。 例如上述例子中，执行upperMesg()方法，将会访问msg变量。

js
function initComputed(vm: Component, computed: Object) {    
    for (const key in computed) {
        const userDef = computed[key];
        const getter = isFunction(userDef) ? userDef : userDef.get; 
        if (!isSSR) {
        // create internal watcher for the computed property.
        watchers[key] = new Watcher(
            vm,
            getter || noop,
            noop,
            computedWatcherOptions)
        }
    }
}

Render Watcher ​

Render Watcher负责收集定义在模板中的响应式变量

js
<template>
  <div id="app">
    {{msg}}
  </div>
</template>
<script>
export default () {
  data () {
    return {
      msg: 'message'
    }
  }
}
</script>

Render Watcher的构造发生在组件的挂载mount过程中。mount函数中会调用mountComponent函数 在mountComponent中创建Watcher的实例。

js
Vue.prototype.$mount = function(
     el?: string | Element,
     hydrating?: boolean): Component {
    return mountComponent(this, el, hydrating);
}

export function mountComponent(
  vm: Component,
  el: Element | null | undefined,
  hydrating?: boolean
): Component { 
    // ...
    updateComponent = () => {
      vm._update(vm._render(), hydrating)
    }
    new Watcher(
        vm,
        updateComponent,
        noop,
        watcherOptions,
        true /* isRenderWatcher */
    );
  // ....

}

Render Watch在创建的过程中，将vm，当前组件实例，作为参数，updateComponent函数作为getter函数传入到Watcher中。 watcher初始化的时候会调用getter函数执行updateComponent。因此下面会将注意力注意到vm._render()和vm._update()函数中。 在这两个函数的执行过程中，会将模板中进行依赖收集。
