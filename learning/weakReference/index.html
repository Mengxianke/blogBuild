深拷贝 ​

ts
const isObject = (obj) => [
    const type = typeof obj;
    return obj !== null && (type === 'object' || type === 'function')
]

const deepClone = (obj) => {
    // 碰触递归边界
    if (!isObject(obj)) {
        return obj;
    };
    // 是对象，我们进一步确定是数组还是{}
    const obj_ = Array.isArray(obj) ? [] : {};
    for (let i in obj) {
        // 访问所有节点，执行深拷贝子问题
        obj_[i] = deepClone(obj[i]);
    };
    // 返回当前深拷贝的obj节点obj_
    return obj_;
}

深拷贝一个对象，需要逐层遍历, 逐层拷贝。将递归过程展开成一颗树，采用深度优先遍历的思想进行拷贝。 dfs深度优先遍历对象中的key。在归的过程中计算。 访问一个对象(局部的某一部分)下的所有key, 执行deepClone方法。 与后续遍历思想，访问左子树，右子树，最后是根节点。自底向上构建拷贝子问题。 在当前节点中，返回当前对象的一个深拷贝。递归边界为，当obj不是对象的时候，递归结束，碰触叶节点。

解决对象间的循环引用现象。当循环引用出现时，子树会无限递归。导致爆栈的现象出现。 保存对象和对象拷贝之间的关系。如果有引用关系出现，直接返回拷贝结果。

ts
// obj.obj = obj; 无限延申的递归树
const deepClone = (obj, map = new Map()) => {

  // 省略边界判断
  // 
  if (map.has(obj)) {
    return map.get(obj);
  };

  const obj_ = .....
  // 存储当前拷贝的对象，以及我们要返回的对象
  map.set(obj, obj_);
  // 省略左右根的访问
};

map数据结构强持有对象，容易阻碍对象的回收。在拷贝的过程中， 将强持有改为弱持有，不阻碍外部对对象的GC。将Map数据结构改成WeakMap()数据结构。 根据WeakMap的性质，当原对象被释放时，对应的拷贝副本也会释放。

ts
const deepClone = (obj, map = new WeakMap()) => { };

事件中心弱持有监听器 ​

朴素版本的订阅与发布设计模式中，EventCenter维护了事件Id与其对应的监听器的对应关系。 该版本容易引发如下问题:

  1. 阻碍外部对监听器的释放，造成内存泄漏现象
  2. 当忘记调用off接口，取消监听，造成监听器一直没有释放。造成内存泄漏。 基于上述的一系列弊端，需要建立起某种机制来防止监听器未释放以及对重复添加进行预警。 源码地址: https://github.com/xiangwenhu/evm。 原作者在订阅与发布中的事件中心的基础上实现了防监听器内存泄漏以及预警机制。 当前版本为简化版本，核心突出弱引用理念。
ts
class EventCenter {
    private events: Map<string, Set<WeakRef<Function>>> = new Map();

    // 通过Set<WeakRef<Function>>合集，找到listener
    getWeakListenerFromHandler(listeners: Set<WeakRef<Function>>, target: Function) {
        for (let listener of listeners) {
            const key = listener.deref();
            if (key && key === target) {
                return listener;
            }
        }
        return null;
    }
    // 监听
    on(eventName, listener) {
        if (!this.events.has(eventName)) {
            this.events.set(eventName, new Set());
        }
        // 构造弱引用
        const weakRefListener = new WeakRef(listener);
        this.events.get(eventName).add(weakRefListener);
        // 创建一个终结，传入listener对应的弱引用
        const fr = new FinalizationRegistry((weakFunc: WeakRef<Function>) => {
            this.events.forEach((listeners, eventName) => {
                listeners.delete(weakFunc);
            });
        });
        // 对listener注册一个终结， 传入listener对应的弱引用
        fr.register(listener, weakRefListener);
    }

    // 取消监听，寻找listener与weakRef<>之间的关系
    off(eventName, listener) {
        const listeners = this.events.get(eventName);
        if (!listeners) {
            return;
        }
        const key = this.getWeakListenerFromHandler(listeners, listener);
        if (key) {
            listeners.delete(key);
        }
    }

}

思考 ​

在设计接口，或者封装工具类函数时,需要考虑一定要强引用外部的变量。 在条件允许的情况下，应当使用若引用机制，保证不会阻碍外部对变量的释放，不会引发内存泄漏问题。
