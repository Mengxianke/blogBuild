loader ​

Loader的作用是一种用于转换模块源代码的工具， 它允许Webpack在导入或加载模块时预处理文件，从而处理各种类型的文件(.vue, .ts)， 并将它们转换成Webpack可以识别的javascript模块。

js
// 导出的javascript模块
module.exports = function (source) {
    // .....
}

Loader配置 ​

Loader的配置两种方案:

webpack配置 ​

js
module.exports = {
    ...
    module: {
        rules: [
        {
            test: /.vue$/,
            loader: 'vue-loader'
        },
        {
            test: /.scss$/,
            use: [
                'style-loader',
                'css-laoder',
                // ....
            ]
        }
        ]
    }
}

内联 ​

webpack允许在引入模块的时候直接执行loader, 这样指定loader称为内联inline loader方式。 import Styles from 'style-loader!css-loader?modules!./styles.css';。 每个loader之前同!隔开，允许携带query参数，最后的模块也使用!隔开

Loader执行顺序 ​

pitch Loader ​

loader的pitch属性叫Pitch Loader

js
loader.pitch = function(remainingRequest,previousRequest,data) {

}

normal Loader ​

loader函数本身叫Normal Loader

js
function (source) {
    // 
    return source;
}

执行顺序 ​

处理阶段 ​

loader处理阶段首先会从左往右经过pitch loader的函数调用，一层一层处理。 它的处理顺序是post, inline, normal, pre。 pitch阶段处理完成后，这一步才会读取引入的资源文件。

执行阶段 ​

将读取到的资源文件交给normal-loader函数，一层一层的处理，它的执行顺序是: pre, normal, inline, post。

熔断机制 ​

在处理阶段，如果pitch loader有返回值的时候。 pitch loader对后序的loader采取熔断措施，不再执行后边的pitch loader。而是会执行其前置的normal-loader, 改变loader chain。 在从右向左依次执行normal-loader。

css loader ​

css-loader的作用

  1. 解析css文件中的@import和url语句
  2. 处理css module, 并将结果作为一个js模块返回
    打包后的模块: 处理css之间的依赖关系:
css
// a.css
@import './b.css'; // 导入b.css

.a {
  font-size: 16px;
}

// b.css
@import './c.css'; // 导入c.css
.b {
  color: red;
}

// c.css
.c {
  font-weight: bolder;
}

style loader ​

style-loader的作用是将CSS代码插入到DOM中。 style-loader是一个pitch loader:

js
// normal loader
module.exports = () => {
    // 无内容
};
// pitch loader
module.exports.pitch = function loader(request) {
    // .....
    // 内联css loader, 使用css-loader对文件进行翻译，获取到DOM内容
    const requestPath = loaderUtils.stringifyRequest(this, '!!' + remainingRequest);
    // example
    // var content = require("!!../../node_modules/css-loader/dist/cjs.js!./global.css");
    return `
    const content = require(${requestPath})

    const style = document.createElement('style');

    style.innerHTML = content;

    document.head.appendChild(style);
  `;
}

采用pitch loader的熔断机制。css-loader返回的是一个js模块， 下一个loader不会感知到上一个loader传递的模块结构。 因此无法直接获取到css样式。需要style-loader通过require('xxxx.css')方法主动获取css内容，再将css内容插入到DOM结构中。 真正的插入到DOM结构是在执行阶段。

file loader ​

解决样式引入路径导致错误的问题。backgorund的url，background: url('../../......')， 是相对原始css文件所在的路径，而非打包后 引入的路径。这就会导致图片引入失败。file-loader 可以解析项目中的 url 引入，根据我们的配置，将图片拷贝到相应的路径，修改打包后文件引用路径，使之指向正确的文件

js
const loaderUtils = require('loader-utils')
module.exports = function (content, map = null, meta = {}) {
    // 是否被url-loader处理过，处理过的话返回base74，url-loader在下面小结具体实现
    const { url,base64 } = meta
    if (url) {
        return `module.exports = "${base64}"`
    } else {
        // 根据当前的上下文，生成一个文件路径，基于dist打包目录，这里生成的文件地址就是：dist/assets/img.jpg
        const interpolateName = loaderUtils.interpolateName(
            this,
            'assets/[name].[contenthash].[ext][query]',
            { content })
        }
        // webpack特有方法，生成一个文件
        this.emitFile(interpolateName, content);
        return `module.exports = "${interpolateName}"`
    }
// 添加标记，表示这是一个raw loader
module.exports.raw = true

url loader ​

  1. 当文件大小<limit参数的时候，url-loader将会把文件转化成内联的base64 uri。图片较多，通过url-loader, 减少http请求，提升页面性能。
  2. 当文件大小>limit参数的时候，url-loader将会把资源的处理交给file-loader，放到配置options下的name属性下。
js
{
    test: /\.(png)|(gif)|(jpg)|(svg)|(bmp)$/i,
    use: [
    {
        loader: 'url-loader',
        options: {
            limit: 1024,
            name: 'assets/images/[name].[hash:5].[ext]',
            esModule: false,
        },
    }],
},
